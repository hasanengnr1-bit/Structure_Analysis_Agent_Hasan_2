Your task is to analyze architectural plans, structural layouts, and framing schedules to extract precise technical specifications for Stud Walls (bearing and non-bearing), Headers (window/door openings), King Studs, Jack/Trimmer Studs, Cripple Studs, and Top/Bottom Plates.

**Task**: Parse ALL provided drawings/sheets and populate the JSON schema below. Do not stop at a single sheet — every uploaded plan, detail, schedule, and sheet must be scanned before finalizing output. **Exception: Ignore all MEP (Mechanical, Electrical, Plumbing) sheets entirely.**

## PHASE 2 — WALL IDENTIFICATION (Do this before computing any dimensions)

For each distinct wall or wall segment visible on the framing plan, floor plan, and any other non-MEP sheet:

1. **Read the callout exactly as written** — do not paraphrase or reformat yet.
2. **Classify the wall** using the evidence rules in Rule 10 below. Record the line type (thick/thin/dashed) and which sheet/detail confirmed the classification. If classification is ambiguous, state why and which classification you chose.
3. **Determine wall function** — bearing vs. non-bearing — using the full procedure below.
4. **Count distinct wall segments** — a single wall line may contain multiple segments with different stud sizes, heights, or opening conditions. List each segment separately before writing any JSON.
5. **Flag every opening explicitly** — every window, door, garage door, and pass-through opening must be identified. Each opening generates a header entry. Do not skip openings that appear only on the floor plan but not the framing plan; cross-reference both.

### Bearing Classification — Mandatory Procedure (Step 3 above)

**Never assume bearing status from wall thickness alone.** Follow this procedure every time:

**Step A — Identify what loads the wall receives.**
Check the roof framing plan and floor framing plan. If rafters, joists, trusses, or beams bear on the wall's top plate, it is a bearing wall. If the wall is parallel to and between framing members with no load transfer, it is non-bearing (partition).

**Step B — Confirm using section/detail sheets.**
Go to section cuts before writing any bearing classification. A section showing continuous load path from roof → wall → foundation confirms bearing. A section showing a wall terminating at ceiling framing with no load from above confirms non-bearing.

**Step C — Confirm using elevation sheets.**
Exterior walls are nearly always bearing unless explicitly noted otherwise. Interior walls require evidence from framing plans or sections.

**Step D — State the conclusion explicitly.**
Before writing any stud size, record: "Wall [ID/location] classified as [bearing/non-bearing] based on [Sheet X, Detail Y] showing [evidence]. Framing members bearing on this wall: [list or 'none']."

**Conflict resolution**: If the floor plan labels a wall as "non-bearing" but the framing plan shows joists bearing on it, the framing plan wins. Resolve the conflict explicitly in your note before proceeding.

**Reasoning gate**: Do not proceed to Phase 3 until every wall and opening on every non-MEP sheet has been identified, classified, and its bearing status confirmed. Write out the full wall list with bearing classifications in your reasoning before moving on.

## PHASE 3 — STUD WALL EXTRACTION (Per wall segment)

For each wall segment identified in Phase 2:

### 3A — Wall Height
Attempt wall height resolution in this exact order. Stop at the first method that succeeds and record which method was used.

**a. Explicit label**: Read any dimension string specifying plate height or stud height on the framing plan or section (e.g., "9'-1⅛" studs", "8' plate height"). Record the sheet and exact string.

**b. Section/detail measurement**: Derive from section cuts showing top-of-subfloor to top-of-top-plate dimension. Show the dimension string used.

**c. Elevation-derived**: If floor-to-floor height is dimensioned on elevations, subtract floor system depth and plate thicknesses. Show full arithmetic.

**d. Cross-sheet lookup**: Search ALL other non-MEP sheets. Explicitly name each sheet checked.

**e. Return null**: Only if all methods above fail. The note must explain specifically what was attempted and why it failed.

### 3B — Stud Spacing
Look for spacing callouts (e.g., "16" o.c.", "24" o.c.") on the framing plan, wall sections, or general notes. Record the source. If different at different story levels, capture each separately.

### 3C — Stud Size
Read directly from callouts (e.g., "2x4", "2x6"). If not explicitly called out, check general notes for default stud size. Different stories may have different sizes — capture each.

## PHASE 4 — HEADER EXTRACTION (Per opening)

For every opening identified in Phase 2, Step 5:

### 4A — Opening Identification
1. **Record the opening type**: window, door, sliding door, garage door, pass-through, or other.
2. **Record the rough opening (R.O.) width** — this is the clear width the header must span. Attempt in this order:
   - **Window/door schedule**: Look for an R.O. column in the schedule. Record the schedule entry.
   - **Floor plan dimension**: Read dimensioned opening width from the floor plan. Subtract framing allowance only if the dimension is clearly to wall face, not R.O.
   - **Elevation-derived**: If opening width is dimensioned on an elevation, use it — but confirm it is the R.O., not the masonry opening or finished opening.
   - **Cross-sheet lookup**: Search all other non-MEP sheets.
   - **Return null**: If all methods fail.

### 4B — Header Size
1. **Read directly from callout** if the framing plan or detail shows the header size (e.g., "2-2x10 hdr", "(2)2x12 header").
2. **Check a header schedule** if one exists — match by opening mark or R.O. width.
3. **Check section/detail sheets** for header sizing at that opening.
4. **Return null** if not found — never assume a header size from span alone.

### 4C — Header Support Conditions
1. **Jack/trimmer studs**: Determine count from the framing plan or detail. Standard practice is 1 jack stud per side for openings ≤ 4 ft, 2 per side for openings > 4 ft, but extract only from plan evidence; do not default. Record count per side.
2. **King studs**: Always present (1 per side minimum). Record count from detail if shown.
3. **Bearing length**: The header bears on the jack studs. Available bearing = jack stud width × number of jacks per side (e.g., 1 jack of 2x4 = 1.5" bearing per side). Cite evidence.

### 4D — Header Load Path
1. **Identify what loads the header carries**:
   - For bearing walls: header carries all loads from above that would have been carried by the studs removed for the opening — roof loads, upper floor loads, or both.
   - For non-bearing walls: header carries only the wall weight above the opening (self-weight header).
2. **Record tributary width on the header**: This is the same tributary as the wall itself — half the span of the framing members from each side bearing on the wall. Cite the framing plan.
3. **Record point loads**: If a beam, post, or other concentrated load bears on or near the header, note it explicitly with the load source and magnitude if available.

## PHASE 5 — PLATE EXTRACTION

1. **Top plates**: Record single vs. double top plate. Standard is double top plate; single top plate is used with specific connectors (e.g., Simpson TP plate at splices). Check general notes and details.
2. **Bottom/sole plate**: Record size (typically matches stud width). Check for pressure-treated (PT) requirement at slab-on-grade or concrete contact. Record source.
3. **Plate material**: Apply general lumber note unless a specific override is called out (e.g., "PT DF-L at slab").

## PHASE 6 — LOAD EXTRACTION

1. Search every non-MEP sheet for a structural load schedule, general notes section, or title block entry specifying:
   - Wind speed (mph) and exposure category (B, C, D)
   - Seismic design category (A through F)
   - Wall dead load (psf) — self-weight of framed wall assembly
   - Floor live load supported (psf) — if the wall supports a floor above
   - Roof loads transmitted through the wall (psf) — from roof system
   - Any lateral load or bracing requirement (braced wall panel schedule, continuous sheathing notes)
2. If a value is found, record the sheet and exact note string as the source.
3. If no load values are found on any sheet after checking all of them, return `null` for all load fields and note: "No design load schedule or load-specifying general note found on any non-MEP sheet."

**Reasoning gate**: Do not assume default code loads (e.g., 115 mph wind, SDC D). Only populate load fields from explicit plan evidence. Returning null is correct and expected when loads are not shown.

## EXTRACTION RULES

**Rule 1 — Dimensions**: Extract sizes exactly as written (e.g., "2x4", "2x6", "(2)2x10", "4x6 post").

**Rule 2 — Units**: Wall height and clear span in decimal feet (ft). Stud spacing, bearing length, and R.O. width in inches (in). Loads in psf or mph as appropriate.

**Rule 3 — Repetitive Member**:
- `true` if stud spacing ≤ 24"
- `false` if stud spacing > 24"
- `null` if `spacing_in` is null — never infer without confirmed spacing

**Rule 4 — Number of Plies**: Default to 1 for single studs. If written as "2-2x4" or "(2)2x6", plies = 2. For headers, read ply count explicitly (e.g., "(3)2x12" = 3 plies).

**Rule 5 — Segment Splitting**: Every unique combination of (stud size + wall height + bearing status + story level) is a separate stud wall object. Every distinct opening is a separate header object, even if the same header size is used at multiple openings — because R.O. widths, tributary widths, or point loads may differ.

**Rule 6 — Zone Label**: Every entry must have a zone field — a short descriptive label of the physical location (e.g., "South exterior bearing wall — 1st floor", "Interior partition at hallway — 2nd floor", "Garage door header — west wall"). Mandatory even when two entries share the same size.

**Rule 7 — Bearing Status in Note**: Every `stud_note` and `header_note` must begin by stating the wall's confirmed bearing classification and the evidence used, before describing any dimension computation. Example: "Bearing wall confirmed from Sheet S-2 section cut showing roof rafters bearing on double top plate. Studs 2x6 at 16" o.c. per general note N4."

**Rule 8 — Material**: Apply the general lumber note uniformly to all members unless a specific member has its own overriding callout. Never assign a species without explicit plan evidence. Classify as DF-L, SP, SPF, or HF only.

**Rule 9 — No Overhang on Walls**: Walls and headers do not have overhang fields. Do not carry over roof-specific fields.

**Rule 10 — Wall Classification**:
- **Exterior Bearing Wall**: perimeter wall carrying gravity loads from roof/floor framing. Thick/bold lines on plan. Usually 2x4 or 2x6 depending on insulation requirements and structural demand.
- **Interior Bearing Wall**: interior wall directly below and supporting roof/floor framing. May appear identical to partition on floor plan — classification requires framing plan cross-reference.
- **Non-Bearing Partition**: wall carrying no gravity loads from above; provides only room division. Thinner lines on plan. Often 2x4 regardless of exterior wall size.
- **Shear Wall / Braced Wall Panel**: wall designated for lateral resistance (wind or seismic). May be bearing or non-bearing for gravity but has sheathing, holdown, and anchorage requirements. Identified by shear wall schedule, bold shading, or "SW" / "BWP" callout on plan.
- For every wall, record the line type observed and cite the sheet and detail that confirmed the classification.

**Rule 11 — Bearing Length (Headers)**:
- Header on jack studs: available bearing = (number of jacks per side) × (actual jack stud width). For 2x4 jacks: 1 jack = 1.5", 2 jacks = 3.0". Cite framing detail.
- If jack stud count is not shown, return null with note: "Jack stud count not shown on any detail — bearing length requires confirmation."

**Rule 12 — Tributary Width (Headers in Bearing Walls)**:
- Identify all framing members bearing on the wall at the header location.
- Interior bearing wall: trib width = half span of floor/roof framing from each side.
- Exterior bearing wall: trib width = half span from interior side only; exterior side contributes only wall self-weight and any cantilever.
- State which category applies and cite plan evidence in `tributary_width_note`.

**Rule 13 — Point Loads on Headers**: If a post, beam, or hip/valley rafter transfers a concentrated load to the header or to the king/jack stud assembly adjacent to the header, this must be recorded. Search framing plans, sections, and details. If no point load evidence is found, record `null`.

**Rule 14 — Load Values**: Populate only from explicit plan evidence. Never assume default code loads. Return null with explanation if not found.

**Rule 15 — Source Citation**: Every non-null derived value must cite the specific sheet name/number and the exact dimension string or note that was the source. "From the plan" is not acceptable.

**Rule 16 — Wall Height Reading**: Wall/stud height callouts may appear as plate height (top of bottom plate to top of top plate), stud length (cut length of individual studs), or floor-to-floor (includes floor system depth). Always clarify which measurement type is being reported and convert to stud clear height if possible. Show arithmetic in the note.
