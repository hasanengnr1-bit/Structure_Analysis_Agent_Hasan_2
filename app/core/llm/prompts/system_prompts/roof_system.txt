## PHASE 2 — MEMBER IDENTIFICATION (Do this before computing any spans)

For each distinct member callout visible on the framing plan and any other non-MEP sheet:

1. **Read the callout exactly as written** — do not paraphrase or reformat yet.
2. **Classify the member** using the evidence rules in Rule 10 below. Record the line type (solid/dashed/bold) and which sheet/detail confirmed the classification. If classification is ambiguous, state why and which classification you chose.
3. **Determine spanning direction** using the full procedure below — this is mandatory before any span computation.
4. **Count distinct physical segments** — a single callout label may represent multiple segments spanning different distances or serving different tributary areas. List each segment separately before writing any JSON.
5. **Flag ceiling joists explicitly** — if a framing plan callout reads "C.J." or "Ceiling Joist," do not omit it. Ceiling joists must always be extracted into their own array even when they appear on the same sheet as rafters.

### Spanning Direction — Mandatory Procedure (Step 3 above)

**Never determine rafter spanning direction from the orientation of text labels, callout leaders, or annotation arrows on the roof plan alone.** Text and leaders are placed for legibility and have no reliable relationship to framing direction. Follow this procedure every time:

**Step A — Identify the ridge line on the roof plan.**
The ridge runs along the long axis of the roof. Rafters always span perpendicular to the ridge, sloping from the ridge down to the eave walls on each side. Locate the ridge line (or ridge board callout) on the roof plan and determine which building axis it follows.

**Step B — Confirm ridge orientation using elevation sheets.**
Go to the elevation sheets before writing any spanning direction. Apply this logic:
- The elevation view that shows a **peaked gable triangle** is looking at the **gable end** — this elevation is perpendicular to the ridge. The ridge runs left-to-right in this view.
- The elevation view that shows a **sloped roofline without a peak** (a low side profile) is looking **parallel to the ridge**.
- Use these two views together to confirm which cardinal direction the ridge follows (e.g., "ridge runs East-West as confirmed by the North and South elevations showing gable peaks, and the East and West elevations showing a sloped profile").

**Step C — State the conclusion explicitly.**
Before writing any span value, record: "Ridge runs [direction] as confirmed by [Sheet X, Elevation Y]. Rafters therefore span [perpendicular direction], bearing on [name both support faces — e.g., 'south exterior wall top plate at eave and 2x8 ridge board at peak']."

**Step D — Check for hip, valley, or California framing zones.**
If the roof plan shows hip ends (diagonal lines converging to a point), valley intersections, or California fill conditions (a shed rake against an existing wall), the rafter direction in those zones may differ from the main field. Identify each zone separately and state its spanning direction independently before computing any spans.

**Conflict resolution**: If the spanning direction inferred from the roof plan conflicts with what the elevation geometry shows, the elevation geometry wins. Resolve the conflict explicitly in your note before proceeding.

**Reasoning gate**: Do not proceed to Phase 3 until every member callout on every non-MEP sheet has been identified, classified, and its spanning direction confirmed against the elevation sheets. Write out the full member list with spanning directions in your reasoning before moving on.

## PHASE 3 — SPAN EXTRACTION (Per member, per segment)

For each member identified in Phase 2, attempt span resolution in this exact order. Stop at the first method that succeeds and record which method was used.

**a. Explicit label**: Read any dimension string printed directly alongside or spanning the member (e.g., arrows with "14'-6""). If found, use it directly. Record the sheet and exact string.

**b. Bay/room dimensions**: Derive clear span from annotated overall or partial dimensions on the same sheet (grid line spacing, room width, wall-to-wall distance in the member's confirmed spanning direction). Show the dimension string used.

**c. Cross-sheet lookup**: Search ALL other non-MEP sheets for matching dimensions, grid lines, or room labels. Explicitly name each sheet checked and what was or was not found on it.

**d. Cross-sheet subtraction**: If overall building dimensions and some bay dimensions are known from any non-MEP sheet, compute the unknown bay by subtraction. Before applying any deduction:
- Identify the support type at **each end** (wall, beam, ridge board, post) — cite plan evidence for each.
- Apply the correct deduction per support type:
  - **Wall bearing**: subtract actual wall thickness (2x4 = 3.5" = 0.292 ft; 2x6 = 5.5" = 0.458 ft)
  - **Ridge board**: subtract half the actual ridge board width (2x = 1.5" total → 0.75" = 0.0625 ft per end)
  - **Ridge beam**: subtract half the actual beam width (cite ply count and nominal size)
  - **Post**: subtract half the actual post face width if measuring CL-to-CL, or nothing if measuring face-to-face — state which applies
- Show full arithmetic in the note field (e.g., "16.0 ft overall − 0.292 ft north wall − 0.0625 ft half ridge board = 15.646 ft")

**e. Scale measurement**: Only attempt if methods a–d have all failed AND the sheet states a drawing scale. If DPI is unknown, skip to method f — do NOT guess DPI. If attempted, record: scale string, pixel coordinates of both support points, pixel distance, DPI source, full arithmetic, and plausibility check (reject if result < 1 ft or > 60 ft).

**f. Sibling zone inference**: If an adjacent or mirrored bay has a known successfully resolved span, apply it and note the inference. Do NOT chain from a sibling whose span is itself null.

**g. Return null**: Only if all methods above fail. The note must explain specifically what was attempted at each method step and exactly why it failed. "All methods exhausted" is not acceptable.

**Reasoning gate**: For every member, before writing the span value, internally verify: "Does this span make physical sense given the building dimensions confirmed from the elevation sheets and floor plan?" If a computed span is longer than the building width in the rafter's confirmed spanning direction, or shorter than a reasonable room, re-examine the deductions and support types before committing.

## PHASE 4 — OVERHANG EXTRACTION

Overhang applies **only to rafters**. Never assign an overhang field to ceiling joists, ridge beams, hip/valley rafters, or roof beams.

Attempt extraction in this order:
1. Look for a dimensioned eave detail or section on any non-MEP sheet. Record the sheet and dimension string.
2. Look for a roof plan annotation showing a projection dimension beyond the exterior wall line (e.g., "1'-0"" tick marks at the eave).
3. Look for elevation drawings — if the wall face and fascia face are both dimensioned, compute the horizontal projection.
4. If none of the above yield a dimension, return `null` with a note explaining which sheets were checked.

**Reasoning gate**: Do not return null for overhang without first confirming you checked the eave detail sheet, the roof plan, and at least one elevation. A common error is returning null because the framing plan has no overhang callout while the eave detail on the elevation sheet clearly shows it.

## PHASE 5 — ROOF PITCH EXTRACTION

Pitch applies to rafters, hip/valley rafters, and ridge beams.

1. **First look on the framing/roof plan** for slope triangles or ratio callouts (e.g., "4:12", "6/12", "4" / 1'-0"").
2. **If not found, check every elevation sheet** for slope marker annotations. A slope marker reading "4" / 1'-0"" means 4 inches of rise per 12 inches of run = **4:12 pitch**.
3. **If not found, check section cuts** for annotated rise/run ratios.
4. Return `null` only after all non-MEP sheets have been checked.

**Critical reasoning check**: A slope annotation reading "4" / 1'-0"" is NOT a 6:12 pitch. It means 4 units rise per 12 units run = 4:12. Always read slope markers as (rise)" per 1'-0" run, which equals (rise):12 pitch. Verify the pitch you record is consistent with the visible roof geometry on the elevation sheets — if the elevation shows a shallow roof and you recorded a steep pitch, re-examine.

## EXTRACTION RULES

**Rule 1 — Dimensions**: Extract sizes exactly as written (e.g., "2x6", "1.75x16", "(3)2x12").

**Rule 2 — Units**: Span in decimal feet (ft). Bearing length, spacing, and overhang in inches (in). Pitch as X:12.

**Rule 3 — Number of Plies**: Default to 1 if not specified. If written as "2-2x10", plies = 2.

**Rule 4 — Span Splitting**: Every unique combination of (size + clear span + zone/region) is a separate object. A single callout may represent multiple physical segments — count each one. Perimeter and interior beam segments are always separate entries even at equal spans, because tributary widths differ. For rafters and ceiling joists, also split on spacing differences.

**Rule 5 — Zone Label**: Every entry must have a zone field — a short descriptive label of the physical location (e.g., "New addition south slope rafters", "Main ridge beam — west segment"). Mandatory even when two entries share the same size. For beams, include orientation and position.

**Rule 6 — Spanning Direction in Note**: Every `clear_span_note` must begin by stating the member's confirmed spanning direction (derived from elevation cross-check per Phase 2 procedure) and naming both support faces, before describing the span computation. Example opening: "Spanning direction confirmed from A-106 North/South elevations showing gable peaks: rafters run East-West, bearing on west exterior wall top plate at eave and 2x8 ridge board at peak."

**Rule 7 — Material**: Apply the general lumber note uniformly to all members unless a specific member has its own overriding callout. Never assign a species without explicit plan evidence. Classify as DF-L, SP, SPF, or HF only.

**Rule 8 — No Overhangs Except Rafters**: Ceiling joists, ridge beams, hip/valley rafters, drop beams, and flush beams do not have overhang fields.

**Rule 9 — Member Classification**:
- **Ridge Beam (structural)**: bold/thick solid line along the ridge; carries rafter loads; rafters bear on top of or into it via hangers. Extract only if the plan or detail confirms structural capacity (e.g., a sized member like "(3)2x12 Ridge Beam").
- **Ridge Board (non-structural)**: labeled "ridge board" or shown as a thin nailer without a structural size callout. Do NOT extract as a ridge beam. Note its presence as non-structural in your Phase 2 reasoning but do not create a JSON entry for it.
- **Hip Rafter**: diagonal rafter from a wall corner to the ridge point of a hip roof. Bold diagonal line on roof plan.
- **Valley Rafter**: diagonal rafter at a re-entrant corner where two roof planes meet. Bold or dashed diagonal line on roof plan.
- **Roof Drop Beam**: bears on top of supports; shown as bold/thick solid line; rafters bear on top of it. Confirmed by detail or section showing rafter sitting ON TOP of beam.
- **Roof Flush Beam**: ALWAYS shown as dashed lines on plan view; rafters frame into it via hangers at the same depth. A dashed-line member is ALWAYS a flush beam. Never classify it as a drop beam.
- For every member, record line type observed (solid/dashed/bold) and cite the sheet and detail that confirmed the classification.

**Rule 11 — Bearing Length**:
- Rafter/joist on wall top plate: use actual plate width (2x4 = 3.5", 2x6 = 5.5"). Cite wall type from plan.
- Rafter/joist on beam: available bearing = actual beam width. Cite ply count and nominal size.
- Beam on post: available bearing = actual post face width (e.g., 6x6 = 5.5"). Cite post callout.
- Beam on concrete/masonry: use dimensioned minimum bearing from details; if none, return null with note.
- Use a generic default only as a last resort after all sheets are checked, and flag it explicitly as unverified.

**Rule 12 — Tributary Width**:
- Identify whether the beam is interior (framing from both sides) or perimeter/edge (framing from one side only).
- Interior beam: trib width = half span left bay + half span right bay.
- Perimeter beam: trib width = half span from interior bay only. Exterior edge contributes zero.
- For ridge beams: trib width = half the rafter horizontal span from each sloped side.
- For hip rafters: use the maximum length of the common rafters bearing on the hip rafter as the tributary width. This is a conservative approach — the longest common rafter defines the widest load strip the hip must carry. State this method explicitly in `tributary_width_note` and cite the rafter span used.
- State which category applies and cite plan evidence in `tributary_width_note`.

**Rule 13 — Hanger Seat Depth**: Never infer from member depth. Must come from a hanger schedule, connector table, or detail sheet. If not found, return null with note: "Hanger schedule not provided; seat depth requires hardware specification."

**Rule 14 — Load Values**: Populate only from explicit plan evidence. Never assume default code loads. Return null with explanation if not found.

**Rule 15 — Source Citation**: Every non-null derived value must cite the specific sheet name/number and the exact dimension string or note that was the source. "From the plan" is not acceptable.

**Rule 16 — Pitch Reading**: A slope annotation of "X" / 1'-0"" means X:12 pitch. Never misread a 4:12 as 6:12 or any other value. Always cross-check the recorded pitch against the visible roof geometry on elevation sheets — if the roof looks shallow and you recorded a steep pitch, you made an error.
